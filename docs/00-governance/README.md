# Governance

This directory defines how Terra Commerce specifications and implementation artifacts are named, versioned, approved, changed, traced, and considered complete.

## Approved governance documents

1. [`document-governance.md`](document-governance.md) — authority, ownership, review, and traceability rules.
2. [`naming-convention.md`](naming-convention.md) — file, document ID, requirement ID, branch, commit, API, event, and database naming.
3. [`versioning-strategy.md`](versioning-strategy.md) — semantic versioning for documents, contracts, applications, and releases.
4. [`dependency-map.md`](dependency-map.md) — upstream and downstream artifact dependencies and impact rules.
5. [`change-control.md`](change-control.md) — standard, breaking, editorial, and emergency change process.
6. [`document-status-lifecycle.md`](document-status-lifecycle.md) — Draft, In Review, Approved, Superseded, Deprecated, and Archived states.
7. [`definition-of-done.md`](definition-of-done.md) — completion criteria for specifications, features, APIs, events, database changes, and releases.

## Current baseline

See [`CURRENT_BASELINE.md`](CURRENT_BASELINE.md) for the current Final PRD, FSD, ADR, and downstream specification baseline.

## Change rule

Future changes follow change control and impact analysis. A change to a locked architecture or implementation decision requires a new ADR.
